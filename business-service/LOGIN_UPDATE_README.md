# 登录系统更新说明

## 📋 更新内容

已将登录验证系统从 Mock 数据改为使用**系统管理数据库**中的真实用户数据。

## ✅ 修改的接口

### 1. POST `/auth/login` - 用户登录
- **之前**: 使用硬编码的 `MOCK_USERS` 数据,密码使用 MD5 加密
- **现在**:
  - 从系统管理数据库查询用户
  - 使用 SHA256 验证密码
  - 验证用户状态(是否被禁用)
  - 获取用户关联的角色
  - 更新最后登录时间和IP

### 2. GET `/user/info` - 获取用户信息
- **之前**: 从 `MOCK_USERS` 返回用户信息
- **现在**: 从系统管理数据库查询并返回真实用户信息

### 3. GET `/auth/codes` - 获取权限代码
- **之前**: 从 `MOCK_CODES` 获取
- **现在**:
  - 超级管理员和 `is_admin=true` 的用户拥有全部权限
  - 其他用户根据角色分配权限

### 4. GET `/menu/all` - 获取用户菜单
- **之前**: 从 `MOCK_MENUS` 获取
- **现在**:
  - 超级管理员和 `is_admin=true` 的用户看到所有菜单
  - 其他用户根据角色分配菜单

## 📝 密码加密方式

- **算法**: SHA256
- **实现**: Python 内置 `hashlib` 库
- **存储**: 密码的 SHA256 哈希值存储在数据库中

## 🔄 与系统管理模块的集成

登录系统现在完全集成了系统管理模块:
- 在"系统管理 > 用户管理"中创建的用户可以直接登录
- 在"系统管理 > 角色管理"中配置的角色会影响用户权限
- 禁用的用户无法登录
- 删除的用户无法登录

## ⚠️ 注意事项

1. **密码长度**: 创建用户时密码最少6位
2. **用户状态**: 只有启用状态的用户才能登录
3. **角色分配**: 新创建的用户需要分配角色才能访问相应功能
4. **首次登录**: 使用默认账号 `admin/admin123` 登录后,建议立即修改密码

## 🔧 故障排查

### 问题: 无法登录
- 检查用户是否存在:`SELECT * FROM users WHERE username='admin'`
- 检查用户状态是否启用:`status=1`
- 检查密码是否正确(使用 Python 测试):
  ```python
  import hashlib
  print(hashlib.sha256('admin123'.encode()).hexdigest())
  ```

### 问题: 登录后看不到菜单
- 检查用户是否分配了角色
- 检查角色是否启用
- 查看后端日志中的角色信息
